## 1. 보안 취약점 (Critical)

### 1.1 API 인증 미적용 [CRITICAL]
- **문제**: 대부분의 API 엔드포인트에 인증 검증이 없음
- **영향 파일**: `/api/companies/route.ts`, `/api/consultations/route.ts`, `/api/documents/route.ts` 등
- **위험도**: 미인증 사용자가 모든 회사, 상담, 문서 데이터에 접근/수정/삭제 가능
- **수정 방향**: 모든 API 라우트에 `supabase.auth.getUser()` 인증 미들웨어 추가

### 1.2 RBAC API 미적용 [CRITICAL]
- **문제**: DB에 `role_permissions` 테이블이 있으나 API에서 권한 검증을 하지 않음
- **영향**: 사이드바 메뉴는 역할 기반으로 필터링되지만, API 직접 호출 시 우회 가능
- **예시**: 일반 사용자가 `/api/admin/permissions` POST 직접 호출로 관리자 설정 변경 가능
- **수정 방향**: API 라우트마다 `permission_key` 검증 로직 추가

### 1.3 JWT Secret 하드코딩 기본값 [HIGH]
- **문제**: `process.env.JWT_SECRET || "default-secret-key"` 패턴 사용
- **영향 파일**: `/api/refresh-token/route.ts`, `/api/verify-code/route.ts`
- **위험도**: 환경변수 미설정 시 공개된 기본키로 JWT 토큰 위조 가능
- **수정 방향**: 환경변수 없으면 에러 throw

### 1.4 파일 업로드 검증 부재 [HIGH]
- **문제**: `/api/consultations/upload/route.ts`에서 파일 타입/크기 검증 없음
- **비교**: `/api/chat/upload/route.ts`는 MIME 타입 화이트리스트 + 100MB 제한 적용됨
- **수정 방향**: 상담 파일 업로드에도 chat과 동일한 검증 로직 적용

### 1.5 클라이언트 제공 Role 파라미터 신뢰 [HIGH]
- **문제**: `/api/documents` GET에서 `role` 값을 URL 파라미터로 받아 필터링
- **영향**: `role=""` 전송 시 모든 문서 접근 가능
- **수정 방향**: 서버에서 세션 기반으로 실제 role 조회

### 1.6 XSS 방지 서버사이드 미적용 [MEDIUM]
- **문제**: `sanitizeHtml()`이 `typeof window === "undefined"` 시 원본 그대로 반환
- **영향 파일**: `/src/lib/sanitize.ts`
- **수정 방향**: 서버사이드에서도 sanitize 적용 (isomorphic-dompurify 사용)

### 1.7 Cron 엔드포인트 보호 부재 [MEDIUM]
- **문제**: `CRON_SECRET` 미설정 시 cron 엔드포인트가 완전히 개방됨
- **수정 방향**: 환경변수 없으면 fail-closed (거부)

## 2. 미구현 기능

### 2.1 DB 스키마에만 존재하는 테이블 (UI 없음)

| 테이블 | 설명 | 상태 |
|--------|------|------|
| `customs_clearances` | 통관 관리 | 스키마만 존재, UI 없음 (`customs_costs`와 별개) |
| `customs_clearance_files` | 통관 파일 | 스키마만 존재, UI 없음 |
| `rnds_develop` | R&D 개발 프로젝트 | 스키마만 존재, 페이지는 있으나 연결 불명확 |
| `rnds_develop_contacts` | R&D 개발 담당자 | 스키마만 존재 |
| `approval_templates` | 결재 템플릿 | 스키마 + 타입 정의는 있으나 UI에서 미사용 |
| `approval_template_lines` | 결재 템플릿 라인 | 동일 |
| `product_aliases` | 제품 별칭 (레거시) | `company_product_aliases`로 대체된 듯하나 잔존 |

### 2.2 상담(Consultations) 미사용 필드

상담 폼에서 다음 스키마 필드들이 입력/표시되지 않음:

**무역/거래 필드:**
- `trade_status` (거래 상태: ordered, shipped, in_transit, arrived 등)
- `order_type` (수출/수입 구분)
- `parent_consultation_id`, `split_number` (분할 배송)
- `order_date`, `oc_number` (OC 번호)
- `expected_completion_date` (예상 완료일)

**배송 필드:**
- `incoterms` (EXW, FOB, CIF 등)
- `lc_number` (신용장 번호)
- `port_of_loading`, `port_of_discharge` (적재항, 양하항)
- `shipping_method`, `shipping_company`, `shipping_carrier_id`

**포장 필드:**
- `packaging_width`, `packaging_height`, `packaging_depth`
- `packaging_type`, `packaging_weight`

**금융 필드:**
- `total_remittance`, `total_remittance_krw`
- `remittance_date`, `exchange_rate`, `currency`
- `product_name`, `specification`, `quantity`, `remarks`

> **참고**: 이 필드들은 overseas 모듈에서 부분적으로 사용되나, 상담 폼 자체와는 통합되지 않음

### 2.3 문서(Documents) 미구현 기능

| 기능 | 상태 | 설명 |
|------|------|------|
| 문서 버전 관리 | 미구현 | 게시판에는 있으나 문서에는 없음 |
| PDF 내보내기 | 미구현 | `window.print()`만 존재, PDF 라이브러리 미통합 |
| 자동 만료 처리 | 미구현 | `valid_until` 경과 시 자동 상태 변경 없음 |
| 일괄 상태 변경 | 미구현 | 문서별 개별 처리만 가능 |
| 문서 보관/소프트 삭제 | 미구현 | 영구 삭제만 가능 |
| `overseas_type` 필드 | 미사용 | DB에 존재하나 코드에서 참조 없음 |
| `currency` 필드 활용 | 부분적 | 해외 모듈에서만 사용, 일반 문서 폼에 미표시 |

### 2.4 결재(Approvals) 미구현 기능

| 기능 | 상태 |
|------|------|
| 결재 템플릿 선택/적용 UI | 미구현 (타입만 정의됨) |

| 결재 SLA/시한 관리 | 미구현 |

| 금액 기반 자동 라우팅 | 미구현 |

## 3. UI/UX 취약점

### 3.1 일관성 문제

| 문제 | 상세 | 영향 |
|------|------|------|
| 삭제 확인 방식 불일치 | 상담 모듈은 커스텀 `DeleteConfirmModal` 사용, 통관비용은 `window.confirm()` 사용 | 사용자 경험 파편화 |
| 토스트 알림 이중 시스템 | `Snackbar` (Material-UI)와 `Toast` (커스텀 컨텍스트) 혼용 | 알림 스타일 불일치 |
| 스켈레톤 vs 스피너 혼용 | 스켈레톤 컴포넌트가 존재하나 다수 테이블에서 스피너만 사용 | 로딩 경험 불균일 |

### 3.2 접근성 (Accessibility) [심각]

- **aria-label**: 전체 코드베이스에서 8개만 발견
- 키보드 내비게이션 패턴 부재
- 포커스 인디케이터 미설정
- role 속성 미사용
- **영향**: 스크린 리더 사용자, 키보드만 사용하는 사용자 접근 불가

### 3.3 폼 관련

- 폼 제출 중 개별 필드 로딩 상태 미표시
- 일부 모달에서 required 표시 불완전
- 저장 실패 시 사용자 입력 유실 가능 (임시저장 없음)

### 3.4 데이터 표시

- 문서 검색이 회사명으로만 가능 (문서 내용, 금액 범위 검색 불가)
- 대시보드 차트에 드릴다운 기능 부재 (집계 → 개별 건 이동)
- 긴 목록에서 무한 스크롤 vs 페이지네이션 혼용

## 4. CRM 기능적 개선점

### 4.1 고객 관리 (Customer Management)

| 현재 상태 | 개선 필요 사항 |
|-----------|--------------|
| 기본 회사/담당자 CRUD | **고객 세그멘테이션/태그** 시스템 없음 |
| 회사명 검색만 가능 | **고급 필터** (업종, 지역, 거래 규모, 최근 접촉일 등) 없음 |
| 개별 회사 상세 페이지 | **고객 360도 뷰** (모든 채널 통합 타임라인) 없음 |
| 수동 상담 기록 | **고객 라이프사이클 스테이지** (리드→전망→고객→이탈) 없음 |
| 중복 체크 = 동일 이름만 | **고급 중복 감지** (유사 이름, 사업자번호, 전화번호) 없음 |
| 담당자 퇴사 관리 있음 | **모회사/자회사 관계** 매핑 없음 |

### 4.2 영업 파이프라인 (Sales Pipeline)

| 현재 상태 | 개선 필요 사항 |
|-----------|--------------|
| 문서 상태 (대기→완료→취소) | **퍼널 시각화** (상담→견적→수주 전환율) 없음 |
| 실적 대시보드 있음 | **매출 예측** (가중치 기반 파이프라인 예측) 없음 |
| 기본 성과 지표 | **Win/Loss 분석** (수주 성공/실패 원인 분석) 없음 |
| 월별 추이 차트 | **딜 속도** (상담→수주 평균 소요일수) 미추적 |
| 견적서 만료 알림 | **자동 팔로업 알림** (설정 가능한 리마인더) 부분적 |

### 4.3 분석/리포트 (Analytics)

| 현재 상태 | 개선 필요 사항 |
|-----------|--------------|
| 매출/매입 추이 | **RFM 분석** (최근성, 빈도, 금액 기반 고객 분류) |
| 거래처별 실적 | **고객 생애가치 (CLV)** 산출 |
| 담당자별 성과 | **팀 벤치마킹** (팀원 간 비교 분석) |
| 일일 업무일지 | **활동 히트맵** (시간대/요일별 활동량) |
| 품목별 분석 | **마진 분석** (품목별/거래처별 마진율) |
| 기본 검색 | **커스텀 리포트 빌더** (사용자 정의 보고서) |

### 4.4 메신저-CRM 연동

| 현재 상태 | 개선 필요 사항 |
|-----------|--------------|
| 독립적 채팅 시스템 | 상담/거래에 **대화방 연동** 없음 |
| 파일 공유 가능 | 결재 문서에 **토론방 연동** 없음 |
| 이모지 반응 가능 | **@멘션 → 알림** 연동 미구현 |
| 읽음 확인 | CRM 데이터 컨텍스트 내 **채팅 타임라인** 없음 |

### 4.5 재고/생산

| 현재 상태 | 개선 필요 사항 |
|-----------|--------------|
| LOT 관리 기본 구현 | LOT 분할 이력 추적 UI 보강 필요 |
| 입출고 관리 | **바코드/QR 스캔** 연동 없음 |
| 발주 권장 | 알고리즘 고도화 (리드타임 반영) 필요 |
| BOM (product_materials) | BOM 기반 **자동 원자재 소요량 산출** UI 미완성 |
| 작업지시서 | **간트 차트** 형태 일정 관리 없음 |

## 우선순위별 액션 아이템

### P0 (즉시 수정 필요)
1. 모든 API 라우트에 인증 미들웨어 추가
2. 서버사이드 RBAC 검증 구현
3. JWT 기본키 제거, 환경변수 필수화
4. 파일 업로드 검증 통일 (상담 파일)
